## Overview

JWT (JSON Web Token) authentication is the primary authentication method for KoroFood's Restaurant Management System. Users authenticate with their email and password to receive a JWT token that authorizes subsequent API requests.

## JWT Token Structure

The system uses JWT tokens with the following structure:

<CodeGroup>
```json Token Claims
{
  "roles": ["ROLE_CLIENTE", "ROLE_ADMIN"],
  "sub": "user@example.com",
  "iat": 1678886400,
  "exp": 1678922400
}
```

```json Decoded Token
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "roles": ["ROLE_CLIENTE"],
    "sub": "user@example.com",
    "iat": 1678886400,
    "exp": 1678922400
  },
  "signature": "HMACSHA256(base64UrlEncode(header) + '.' + base64UrlEncode(payload), secret)"
}
```
</CodeGroup>

### Token Fields

<ResponseField name="roles" type="array" required>
  Array of role strings assigned to the user (e.g., `ROLE_CLIENTE`, `ROLE_ADMIN`, `ROLE_RECEPCIONISTA`)
</ResponseField>

<ResponseField name="sub" type="string" required>
  Subject of the token - the user's email address
</ResponseField>

<ResponseField name="iat" type="number" required>
  Issued At timestamp in Unix epoch format
</ResponseField>

<ResponseField name="exp" type="number" required>
  Expiration timestamp in Unix epoch format (10 hours after issuance)
</ResponseField>

## Login Endpoint

Authenticate a user and receive a JWT token.

### Request

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "correo": "user@example.com",
    "clave": "password123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:8080/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    correo: 'user@example.com',
    clave: 'password123'
  })
});

const data = await response.json();
console.log('Token:', data.token);
```

```python Python
import requests

url = 'http://localhost:8080/auth/login'
data = {
    'correo': 'user@example.com',
    'clave': 'password123'
}

response = requests.post(url, json=data)
token = response.json()['token']
print(f'Token: {token}')
```
</CodeGroup>

<ParamField path="clave" type="string" required>

## Implementation Details

The login process is implemented in the `AuthController`:

```java controller/AuthController.java
@PostMapping("/login")
public ResponseEntity<?> loginUsuario(@RequestBody LoginRequest loginRequest) {
    try {
        // Authenticate user credentials
        Authentication auth = authenticationManager.authenticate(
                new UsernamePasswordAuthenticationToken(loginRequest.getCorreo(),
                        loginRequest.getClave()));

        UserDetails userDetails = (UserDetails) auth.getPrincipal();

        // Extract user roles
        List<String> roles = userDetails.getAuthorities().stream()
                .map(GrantedAuthority::getAuthority)
                .collect(Collectors.toList());

        // Generate JWT token
        String token = jwtUtil.generateToken(loginRequest.getCorreo(), roles);
        return ResponseEntity.ok(Map.of("token", token));
    } catch (Exception e) {
        return ResponseEntity.status(401)
                .body(Map.of("error", "Credenciales inválidas"));
    }
}
```

## Token Generation

Tokens are generated by the `JwtUtil` class using the JJWT library:

```java util/JwtUtil.java
@Component
public class JwtUtil {
    
    private final Key key = Keys.hmacShaKeyFor("SK_443ff638f8a74ba659cdb8336bfb5f91".getBytes());
    
    public String generateToken(String username, List<String> roles){
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", roles);
        
        return Jwts.builder()
                .setClaims(claims)
                .setSubject(username)
                .setIssuedAt(new Date(System.currentTimeMillis()))
                .setExpiration(new Date(System.currentTimeMillis() + 1000 * 60 * 60 * 10))
                .signWith(key, SignatureAlgorithm.HS256)
                .compact();
    }
}
```

<Warning>
  The secret key shown in the code example is for demonstration purposes. In production, use a secure secret key stored in environment variables.
</Warning>

## Token Validation

The `JwtUtil` class also provides methods for validating tokens:

```java util/JwtUtil.java
public String obtenerUsuarioAndToken(String token){
    return Jwts.parser()
            .verifyWith((SecretKey) key)
            .build()
            .parseSignedClaims(token)
            .getPayload()
            .getSubject();
}

public boolean validarToken(String token){
    try {
        Jwts.parser().verifyWith((SecretKey) key).build().parseSignedClaims(token);
        return true;
    } catch (Exception e) {
        return false;
    }
}
```

<Note>
  While these validation methods exist in the User Service, token validation is primarily performed by the API Gateway before routing requests to backend services.
</Note>

## Using the Token

Once you have the JWT token, include it in the `Authorization` header for authenticated requests:

<CodeGroup>
```bash cURL
curl -X GET http://localhost:8080/auth/me \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..."
```

```javascript JavaScript
const response = await fetch('http://localhost:8080/auth/me', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const user = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {token}'
}

response = requests.get('http://localhost:8080/auth/me', headers=headers)
user = response.json()
```
</CodeGroup>

### Request

```bash
GET /auth/me
Headers:

```

<Note>
  The API Gateway extracts the username from the JWT token and passes it in the `X-Username` header to backend services.
</Note>

## Best Practices

<Steps>
  <Step title="Store Securely">
    Store JWT tokens securely in your application (e.g., httpOnly cookies or secure storage)
  </Step>
  <Step title="Handle Expiration">
    Implement token refresh logic or redirect users to login when tokens expire
  </Step>
  <Step title="Use HTTPS">
    Always transmit tokens over HTTPS in production to prevent interception
  </Step>
  <Step title="Validate on Gateway">
    The API Gateway validates all tokens before routing to backend services
  </Step>
</Steps>

## Error Handling

<ResponseField name="401 Unauthorized">
  Returned when credentials are invalid or token has expired
</ResponseField>

<ResponseField name="404 Not Found">
  Returned when user account does not exist (GET /auth/me)
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Returned when an unexpected error occurs during authentication
</ResponseField>
