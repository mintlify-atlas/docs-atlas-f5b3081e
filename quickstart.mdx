## Prerequisites

Before you begin, ensure you have the following installed:
- **Docker** (20.10 or later)
- **Docker Compose** (2.0 or later)
- **Git**

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the KoroFood repository from GitHub:

    ```bash
    git clone https://github.com/CoDeVeck/KoroFood-s.git
    cd KoroFood-s
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the project root with your API keys and secrets:

    ```env
    # JWT Authentication
    JWT_SECRET=your_jwt_secret_here

    # Cloudinary (Image Storage)
    CLOUDINARY_NAME=your_cloudinary_name
    CLOUDINARY_KEY_KOROFOODS=your_cloudinary_key
    CLOUDINARY_SECRET_KOROFOODS=your_cloudinary_secret

    # AI Services
    GEMINI_API_KEY=your_gemini_api_key
    GOOGLE_VISION_API_KEY=your_google_vision_api_key

    # Email Configuration
    EMAIL_USERNAME=your_email@example.com
    EMAIL_PASSWORD=your_email_app_password

    # Twilio (SMS Notifications)
    TWILIO_ACCOUNT_SID=your_twilio_account_sid
    TWILIO_AUTH_TOKEN=your_twilio_auth_token
    TWILIO_PHONE_NUMBER=your_twilio_phone_number
    ```

    <Note>
      For development, you can use placeholder values for optional services like Gemini and Google Vision.
    </Note>
  </Step>

  <Step title="Start the Services">
    Launch all microservices and infrastructure with Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This will start:
    - 9 microservices (User, Order, Payment, Event, Menu, Table, Reservation, Qualification, User SOAP)
    - API Gateway (port 8098)
    - Eureka Server (port 8761)
    - PostgreSQL (port 5433)
    - MongoDB (port 27017)
    - Redis (port 6379)
    - RabbitMQ (port 5672)
  </Step>

  <Step title="Verify the Services">
    Check that all services are running:

    ```bash
    docker-compose ps
    ```

    Wait for all services to be healthy. You can monitor the logs:

    ```bash
    docker-compose logs -f
    ```
  </Step>

  <Step title="Access the Eureka Dashboard">
    Open your browser and navigate to:

    ```
    http://localhost:8761
    ```

    You should see all microservices registered and available.
  </Step>
</Steps>

## Your First API Call

Once the services are running, you can make your first API call through the API Gateway:

<CodeGroup>

```bash cURL
# Register a new user
curl -X POST http://localhost:8098/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "John",
    "apellido": "Doe",
    "correo": "john.doe@example.com",
    "clave": "securePassword123",
    "telefono": "1234567890",
    "direccion": "123 Main St"
  }'
```

```javascript JavaScript
// Login and get JWT token
const response = await fetch('http://localhost:8098/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    correo: 'john.doe@example.com',
    clave: 'securePassword123'
  })
});

const { token } = await response.json();
console.log('JWT Token:', token);
```

```python Python
import requests

# Get list of events
response = requests.get('http://localhost:8098/eventos')
events = response.json()
print(f"Found {len(events)} events")
```

</CodeGroup>

## Common Operations

### View Available Events

```bash
curl http://localhost:8098/eventos
```

### Create a Reservation

```bash
curl -X POST http://localhost:8098/reservacion/register \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "idUsuario": 1,
    "idMesa": 1,
    "fecha": "2024-12-25",
    "hora": "19:00:00",
    "cantidadPersonas": 4
  }'
```

### Get Menu

```bash
curl http://localhost:8098/menu/download -o menu.pdf
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Services not starting">
    Check Docker logs for errors:
    ```bash
    docker-compose logs [service-name]
    ```
    
    Ensure all required environment variables are set in your `.env` file.
  </Accordion>

  <Accordion title="Database connection errors">
    Wait for PostgreSQL to fully initialize (about 30-60 seconds on first start). Check health:
    ```bash
    docker-compose ps postgres
    ```
  </Accordion>

  <Accordion title="Port conflicts">
    If ports are already in use, stop conflicting services or modify port mappings in `docker-compose.yml`.
  </Accordion>

  <Accordion title="Authentication errors">
    Ensure your `JWT_SECRET` is set in the `.env` file and the API Gateway is running.
  </Accordion>
</AccordionGroup>
